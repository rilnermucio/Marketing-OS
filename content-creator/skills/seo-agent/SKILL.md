```
CRAWLING → RENDERING → INDEXING → RANKING → SERVING
    ↓          ↓           ↓          ↓          ↓
Googlebot   JavaScript   Database    200+      User
Discovery   Processing   Storage    Signals   Interface
```

| Componente | Função | Impacto SEO |

| **Caffeine** | Índice web principal | Velocidade de indexação |


#### Taxonomia de Search Intent

| Tipo | Descrição | Exemplo | Formato Ideal |
|------|-----------|---------|---------------|
| **Informacional** | Aprender algo | "como fazer SEO" | Guia, tutorial, listicle |
| **Navegacional** | Ir a um site | "login facebook" | Homepage, login page |
| **Comercial** | Pesquisar antes de comprar | "melhor ferramenta SEO" | Comparativo, review |
| **Transacional** | Comprar/converter | "comprar Semrush" | Landing page, produto |

#### Micro-Intenções Dentro de Cada Categoria

**Informacional Expandido:**
```
HOW-TO        → "como otimizar meta description"
WHAT-IS       → "o que é keyword cannibalization"
WHY           → "por que SEO demora"
COMPARISON    → "SEO vs SEM diferenças"
LIST          → "melhores práticas SEO 2025"
DEFINITION    → "definição de backlink"
```

**Comercial Expandido:**
```
BEST          → "melhor ferramenta keyword research"
REVIEW        → "Ahrefs review completo"
VS            → "Semrush vs Ahrefs"
TOP           → "top 10 plugins SEO WordPress"
ALTERNATIVE   → "alternativas ao Moz"
FOR [PERSONA] → "SEO tools para iniciantes"
```

### 2.2 Análise de SERP para Determinar Intenção

#### Sinais da SERP

| Elemento na SERP | Indica Intenção |
|------------------|-----------------|
| Knowledge Panel | Informacional/Navegacional |
| Shopping Results | Transacional |
| People Also Ask | Informacional |
| Local Pack | Local + Transacional |
| Video Carousel | Visual/Tutorial |
| Featured Snippet | Informacional |
| Product Reviews | Comercial |
| Ads dominando | Transacional |

#### Framework de Análise de SERP

```
PARA CADA KEYWORD:
1. Google a keyword (incógnito)
2. Analise os 10 primeiros resultados
3. Identifique padrões:
   - Tipo de conteúdo (artigo, produto, vídeo)
   - Formato (listicle, guia, landing)
   - Profundidade (curto, médio, longo)
   - Elementos (imagens, vídeos, tabelas)
4. Deduza a intenção dominante
5. Crie conteúdo que satisfaça a intenção
```

### 2.3 Matching Intent: A Arte da Correspondência

#### Regra de Ouro

> **Se seu conteúdo não corresponde à intenção de busca, nenhuma quantidade de links ou otimização técnica vai rankear.**

#### Checklist de Intent Match

```markdown
[ ] Analisei os top 10 resultados para a keyword?
[ ] Identifiquei o tipo de conteúdo dominante?
[ ] Meu conteúdo tem formato similar?
[ ] Minha profundidade é igual ou maior?
[ ] Satisfaço a pergunta implícita do usuário?
[ ] Ofereço algo único (angle, dados, formato)?
```

### 3.2 Metodologia de Keyword Research

#### Processo Completo em 7 Etapas

```
1. SEED KEYWORDS
   └── Brainstorm inicial baseado no negócio

2. EXPANSÃO
   ├── Ferramentas (Ahrefs, Semrush, Ubersuggest)
   ├── Google Suggest
   ├── People Also Ask
   ├── Related Searches
   └── Competitor Analysis

3. FILTRAGEM
   ├── Volume mínimo relevante
   ├── Dificuldade viável
   └── Intenção alinhada

4. CLUSTERING
   ├── Agrupar por tópico
   ├── Identificar pillar pages
   └── Mapear cluster content

5. PRIORIZAÇÃO
   ├── Opportunity Score
   ├── Business Value
   └── Quick Wins vs Long-term

6. MAPEAMENTO
   ├── URL existente ou nova
   ├── Tipo de conteúdo
   └── Responsável

7. MONITORAMENTO
   ├── Rankings iniciais
   ├── Posição target
   └── Timeline de revisão
```

### 3.3 Métricas de Keywords

#### Métricas Principais

| Métrica | O Que Mede | Fonte |
|---------|------------|-------|
| **Search Volume** | Buscas mensais médias | Ahrefs, Semrush |
| **Keyword Difficulty** | Competição | Ahrefs (0-100) |
| **CPC** | Valor comercial | Google Ads |
| **CTR Opportunity** | Cliques orgânicos vs pagos | SERP Features |
| **Traffic Potential** | Tráfego real estimado | Ahrefs |
| **Trend** | Sazonalidade/crescimento | Google Trends |

#### Fórmula de Priorização

```
OPPORTUNITY SCORE = (Volume × CTR Opportunity × Business Value)
                    ÷ (Keyword Difficulty × Competition)

Onde:
- Volume: Search Volume mensal
- CTR Opportunity: % de cliques para orgânicos (1 - ads - features)
- Business Value: 1-5 (alinhamento com negócio)
- Keyword Difficulty: 0-100
- Competition: 1-5 (força dos competidores)
```

### 3.5 Keyword Gap Analysis

#### Processo de Análise de Gap

```
1. LISTAR COMPETIDORES
   └── 3-5 competidores diretos

2. EXTRAIR KEYWORDS DOS COMPETIDORES
   └── Ahrefs/Semrush Organic Keywords

3. COMPARAR COM SUAS KEYWORDS
   └── Site Explorer > Content Gap

4. IDENTIFICAR GAPS
   ├── Keywords que competidores rankeiam
   └── Que você não rankeia

5. FILTRAR OPORTUNIDADES
   ├── Volume relevante
   ├── Dificuldade viável
   └── Alinhamento com negócio

6. CRIAR PLANO DE CONTEÚDO
   └── Priorizar por opportunity score
```

### 3.6 Entidades e SEO Semântico

#### O Que São Entidades

> Entidades são "coisas" identificáveis: pessoas, lugares, conceitos, produtos. O Google constrói um Knowledge Graph de entidades e suas relações.

#### Como Trabalhar com Entidades

```
1. IDENTIFICAR ENTIDADES DO NICHO
   └── Use Google Knowledge Graph API

2. MAPEAR RELAÇÕES
   └── Entidade A → Relacionada com → Entidade B

3. INCLUIR ENTIDADES NO CONTEÚDO
   ├── Menção natural de entidades relacionadas
   ├── Schema markup para entidades
   └── Internal links para páginas de entidades

4. CONSTRUIR AUTORIDADE TÓPICA
   └── Cobrir todas as entidades de um tópico
```

#### Exemplo de Mapa de Entidades

```
TÓPICO: "SEO"
    │
    ├── ENTIDADES RELACIONADAS
    │   ├── Google (empresa)
    │   ├── PageRank (conceito)
    │   ├── Backlink (conceito)
    │   ├── Search Console (produto)
    │   └── John Mueller (pessoa)
    │
    ├── CONCEITOS ASSOCIADOS
    │   ├── Ranking
    │   ├── Indexação
    │   ├── Crawling
    │   └── SERP
    │
    └── TÓPICOS RELACIONADOS
        ├── Marketing Digital
        ├── SEM
        ├── Content Marketing
        └── Link Building
```

### 4.1 Arquitetura de Página Otimizada

#### Estrutura HTML Ideal

```html
<!DOCTYPE html>
<html lang="pt-BR">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">

    <!-- TITLE: 50-60 caracteres, keyword no início -->
    <title>Keyword Principal: Benefício ou Contexto | Brand</title>

    <!-- META DESCRIPTION: 150-160 caracteres, CTA incluído -->
    <meta name="description" content="Descrição persuasiva com keyword
    natural. Explica o que o usuário encontrará. Inclui call-to-action.">

    <!-- CANONICAL: Evita duplicação -->
    <link rel="canonical" href="https://site.com/url-limpa/">

    <!-- OPEN GRAPH -->
    <meta property="og:title" content="Título para Social">
    <meta property="og:description" content="Descrição para compartilhamento">
    <meta property="og:image" content="https://site.com/imagem-1200x630.jpg">
    <meta property="og:url" content="https://site.com/url/">
    <meta property="og:type" content="article">

    <!-- TWITTER CARD -->
    <meta name="twitter:card" content="summary_large_image">
    <meta name="twitter:title" content="Título para Twitter">
    <meta name="twitter:description" content="Descrição para Twitter">
    <meta name="twitter:image" content="https://site.com/imagem.jpg">

    <!-- SCHEMA MARKUP -->
    <script type="application/ld+json">
    {
      "@context": "https://schema.org",
      "@type": "Article",
      ...
    }
    </script>

    <!-- HREFLANG (se multilíngue) -->
    <link rel="alternate" hreflang="pt-BR" href="https://site.com/pagina/">
    <link rel="alternate" hreflang="en" href="https://site.com/en/page/">

</head>
<body>
    <header>
        <nav aria-label="Navegação principal">
            <!-- Breadcrumbs com Schema -->
        </nav>
    </header>

    <main>
        <article>
            <!-- H1: Única por página, contém keyword -->
            <h1>Keyword Principal no H1 da Página</h1>

            <p>Parágrafo introdutório que responde à pergunta

            <!-- ÍNDICE: Para conteúdos longos -->
            <nav aria-label="Índice do artigo">
                <h2>Neste artigo</h2>
                <ul>
                    <li><a href="#secao1">Seção 1</a></li>
                    <li><a href="#secao2">Seção 2</a></li>
                </ul>
            </nav>

            <!-- CONTEÚDO: H2s para seções principais -->
            <section id="secao1">
                <h2>Subtópico com Keyword Variação</h2>
                <p>Conteúdo da seção...</p>

                <!-- H3s para subseções -->
                <h3>Ponto Específico</h3>
                <p>Detalhes...</p>
            </section>

            <!-- ELEMENTOS RICH -->
            <figure>
                <img src="imagem.jpg"
                     alt="Descrição detalhada da imagem com contexto"
                     loading="lazy"
                     width="800"
                     height="600">
                <figcaption>Legenda explicativa</figcaption>
            </figure>

            <!-- TABELAS -->
            <table>
                <caption>Título da Tabela</caption>
                <thead>...</thead>
                <tbody>...</tbody>
            </table>

            <!-- LISTAS -->
            <ul>
                <li>Item com keyword semântica</li>
            </ul>

            <!-- FAQ: Para Featured Snippets -->
            <section>
                <h2>Perguntas Frequentes</h2>
                <details>
                    <summary>Pergunta com keyword?</summary>
                    <p>Resposta concisa e direta.</p>
                </details>
            </section>

        </article>
    </main>

    <!-- INTERNAL LINKS: Seção relacionados -->
    <aside>
        <h2>Artigos Relacionados</h2>
```

### 4.2 Otimização de Title Tags

#### Anatomia do Title Perfeito

```
ESTRUTURA: [Keyword Principal]: [Benefício/Modificador] | [Brand]

EXEMPLOS:
✓ "SEO para Iniciantes: Guia Completo 2025 | Moz"
✓ "Como Fazer Keyword Research em 7 Passos | Ahrefs"
✓ "Melhores Ferramentas SEO Grátis (Testadas) | Neil Patel"

ANTI-PATTERNS:
✗ "Bem-vindo ao nosso blog sobre SEO" (sem keyword, genérico)
✗ "SEO SEO SEO Ferramentas SEO Grátis" (keyword stuffing)
✗ "Artigo sobre Search Engine Optimization para..." (muito longo)
```

#### Fórmulas de Title por Tipo de Conteúdo

| Tipo | Fórmula | Exemplo |
|------|---------|---------|
| **How-to** | Como [Ação] [Resultado] em [Tempo/Passos] | "Como Dobrar Tráfego Orgânico em 6 Meses" |
| **Listicle** | [Número] [Adjetivo] [Keyword] para [Ano/Persona] | "15 Melhores Ferramentas SEO para 2025" |
| **Guide** | [Keyword]: Guia [Adjetivo] [Ano] | "Link Building: Guia Definitivo 2025" |
| **Review** | [Produto] Review: [Veredicto] [Ano] | "Ahrefs Review: Vale a Pena em 2025?" |
| **VS** | [A] vs [B]: Qual [Melhor/Diferença] | "Semrush vs Ahrefs: Qual Escolher?" |
| **Template** | [Número] Templates de [Keyword] [Grátis] | "10 Templates de Meta Description Grátis" |

#### Testes de Title

```
CHECKLIST:
[ ] Keyword principal nos primeiros 30 caracteres?
[ ] Total entre 50-60 caracteres?
[ ] Inclui número ou power word?
[ ] Tem benefício claro ou curiosidade?
[ ] Diferenciado dos concorrentes?
[ ] Evita clickbait que não entrega?
```

#### Estrutura Correta

```
<h1>Título Principal (ÚNICO por página)

    ├── <h2>Seção Principal 1

    │
    └── <h2>FAQ / Conclusão
```

#### Regras de Headings

| Regra | Descrição |
|-------|-----------|
| **Um H1 por página** | Sempre, sem exceção |
| **Não pular níveis** | H1 → H2 → H3 (não H1 → H3) |
| **Descritivos** | Devem fazer sentido isolados |
| **Keywords naturais** | Inclua variações, não force |
| **Estrutura lógica** | Outline deve fazer sentido |

### 4.5 Otimização de Conteúdo

#### Densidade de Keyword (Conceito Moderno)

> **A "densidade de keyword" como métrica morreu.** Use keywords naturalmente e foque em:

```
KEYWORD PRINCIPAL:
- No título (H1)
- Nos primeiros 100 palavras
- Em pelo menos um H2
- Na URL
- Na meta description
- No alt text de imagem relevante

KEYWORDS SECUNDÁRIAS/LSI:
- Distribuídas naturalmente
- Em H2s e H3s
- No corpo do texto
- Variações e sinônimos
```

#### TF-IDF e Otimização Semântica

```
TF-IDF = Mede relevância de termos no documento vs corpus

COMO USAR:
1. Analise top 10 resultados para a keyword
2. Extraia termos comuns (use Clearscope, Surfer, Frase)
3. Inclua termos semanticamente relevantes
4. Mantenha naturalidade

FERRAMENTAS:
- Surfer SEO
- Clearscope
- Frase
- MarketMuse
```

#### Comprimento de Conteúdo por Tipo

| Tipo de Conteúdo | Palavras Recomendadas | Justificativa |
|------------------|----------------------|---------------|
| **Blog post informativo** | 1.500 - 2.500 | Profundidade adequada |
| **Guia definitivo** | 3.000 - 7.000+ | Cobertura completa |
| **Landing page** | 500 - 1.500 | Foco em conversão |
| **Página de produto** | 300 - 1.000 | Specs + benefícios |
| **FAQ page** | 1.000 - 2.000 | Responda tudo |
| **News/Atualização** | 300 - 800 | Direto ao ponto |

### 4.6 Otimização de Imagens

#### Checklist de Imagem Otimizada

```markdown
NOME DO ARQUIVO:
[ ] Descritivo (keyword-descricao.jpg, não IMG_4532.jpg)
[ ] Hífen entre palavras
[ ] Lowercase

ALT TEXT:
[ ] Descreve a imagem para quem não vê
[ ] Inclui keyword se natural
[ ] 125 caracteres ou menos
[ ] Não começa com "Imagem de..."

TÉCNICO:
[ ] Formato correto (WebP > AVIF > JPG > PNG)
[ ] Comprimido (TinyPNG, Squoosh, ShortPixel)
[ ] Dimensões definidas (width/height no HTML)

[ ] Responsive (srcset)

SCHEMA:
[ ] ImageObject markup se relevante
[ ] Licença especificada se aplicável
```

### 4.7 Internal Linking Strategy

#### Princípios de Internal Linking

```
1. HIERARQUIA
   Homepage → Categorias → Subcategorias → Artigos
   (PageRank flui de cima para baixo)

2. RELEVÂNCIA
   Linke para páginas tematicamente relacionadas
```

### 5.3 Core Web Vitals

#### Os Três Métricas

| Métrica | Mede | Bom | Precisa Melhoria | Ruim |
|---------|------|-----|-----------------|------|
| **LCP** | Largest Contentful Paint | ≤2.5s | 2.5-4s | >4s |
| **INP** | Interaction to Next Paint | ≤200ms | 200-500ms | >500ms |
| **CLS** | Cumulative Layout Shift | ≤0.1 | 0.1-0.25 | >0.25 |

#### Otimização de LCP

```
CAUSAS COMUNS DE LCP RUIM:
1. Servidor lento (TTFB alto)
2. Recursos bloqueadores de renderização
3. Imagens grandes não otimizadas
4. JavaScript pesado

SOLUÇÕES:
├── CDN para assets estáticos
├── Preload de imagem LCP
│   <link rel="preload" as="image" href="hero.webp">
├── Otimizar imagens (WebP, dimensões corretas)
├── Lazy load de imagens abaixo da dobra
├── Minimizar CSS crítico inline
└── Defer JavaScript não-crítico
```

#### Otimização de INP

```
CAUSAS COMUNS DE INP RUIM:
1. JavaScript pesado no main thread
2. Event handlers lentos
3. Rendering demorado
4. Third-party scripts

SOLUÇÕES:
├── Code splitting
├── Web Workers para tarefas pesadas
├── Debounce/throttle de event handlers
├── Virtualização de listas longas
├── Lazy load de componentes
└── Auditar third-party scripts
```

#### Otimização de CLS

```
CAUSAS COMUNS DE CLS RUIM:
1. Imagens sem dimensões
2. Ads/embeds sem espaço reservado
3. Fonts carregando tardiamente (FOUT)
4. Conteúdo injetado dinamicamente

SOLUÇÕES:
├── Sempre definir width/height em imagens
│   <img src="..." width="800" height="600">
├── aspect-ratio em CSS para containers
├── font-display: swap + preload de fonts
├── Placeholder para ads/embeds
└── min-height para conteúdo dinâmico
```

#### Content Decay e Refresh

```
SINAIS DE QUE PRECISA ATUALIZAR:

MÉTRICAS:
- Queda de rankings (>5 posições)
- Queda de tráfego (>20%)
- CTR diminuindo
- Bounce rate aumentando

CONTEÚDO:
- Informações desatualizadas
- Links quebrados
- Screenshots antigos
- Dados de anos anteriores
- Competidores ultrapassaram
```

#### Processo de Atualização

```
1. IDENTIFICAR
   └── Content audit regular (trimestral)

2. PRIORIZAR
   └── Maior potencial de recuperação primeiro

3. ANALISAR
   ├── O que mudou na SERP?
   ├── O que competidores fizeram?
   └── O que está faltando?

4. ATUALIZAR
   ├── Dados e estatísticas
   ├── Screenshots e imagens
   ├── Seções novas se necessário
   ├── Remover conteúdo obsoleto
   └── Melhorar profundidade

5. RE-OTIMIZAR
   ├── Title e meta description
   ├── Internal links
   └── Schema markup

6. REPUBLICAR
   ├── Atualizar data de publicação
   └── Promover novamente
```

### 8.4 Content Pruning

#### Quando Remover/Consolidar

```
CANDIDATOS A PRUNING:

1. THIN CONTENT
   - Menos de 300 palavras
   - Sem valor único
   - Sem tráfego

2. CONTEÚDO DUPLICADO/SIMILAR
   - Múltiplos artigos sobre o mesmo tema
   - Canibalização de keywords

3. CONTEÚDO OBSOLETO
   - Tecnologias descontinuadas
   - Eventos passados sem relevância
   - Produtos que não existem

4. LOW PERFORMERS
   - Sem tráfego por 12+ meses
   - Sem backlinks
   - Sem conversões
```

#### Processo de Pruning

```
OPÇÕES:

1. REDIRECT 301
   Quando: Conteúdo similar melhor existe
   Como: 301 para a versão canônica

2. CONSOLIDAR
   Quando: Múltiplos artigos thin sobre tema
   Como: Combinar em um artigo completo, 301 os outros

3. ATUALIZAR
   Quando: Bom tópico, execução ruim
   Como: Reescrever completamente

4. NOINDEX
   Quando: Necessário para site, não para busca
   Como: meta robots noindex

5. DELETAR
   Quando: Sem valor nenhum
   Como: Retornar 404/410
```
